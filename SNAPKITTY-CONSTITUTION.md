# SNAPKITTY-CONSTITUTION.md — SnapKitty Constitutional Principles

## Core Principles

1. **Non-recursion**: No operation may call itself directly or indirectly
2. **Typed I/O**: Every stage has explicit input and output types
3. **Deterministic**: Same input always produces same output
4. **WORM Sealed**: Every artifact is write-once, read-many
5. **Fail-Closed**: Any error terminates processing
6. **Observable**: Every intermediate state is inspectable
7. **Sovereign**: Every agent controls its own identity and keys
8. **Governed**: Every action passes through policy before execution
9. **Verified**: Every declaration is formally verified
10. **Audited**: Every decision is recorded in the Archivum

## Architecture

- **Fontana DSL**: Symbolic declaration language
- **AST Parser**: Non-recursive parser
- **Admissibility Validator**: Staged traversal validation
- **Stratum Boundary**: Stratum level verification
- **Contractivity Engine**: Mathematical soundness proofs
- **SnapKitty Governance**: Policy enforcement
- **Lean Verification**: Formal proof checking
- **Rust Runtime**: Deterministic execution
- **WORM Witness**: Artifact sealing
- **Archivum Ledger**: Persistent audit log
- **SnapKitty Observatory**: Runtime telemetry

## Seal

```
SNAPKITTY-CONSTITUTION-001
Principles before code.
Governance before action.
Verification before execution.
No recursion. No exceptions.
```
